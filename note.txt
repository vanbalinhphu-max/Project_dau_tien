Repository (Repo - kho chứa)

- Local Repository: nằm trực tiếp trên máy, sẽ nằm trong folder .git
- Remote Repository: kho lưu trữ trên 1 máy chru khác (Github, Gitlab)




Commit
- là 1 bản sao lưu (snapshot) trạng thái của tất cả các file trong dự án tại MỘT THỜI ĐIỂM CỤ THỂ



Khu vực làm việc (Working Area)


- working directory (thư mục làm việc) 
- staging area (khu vực chuẩn bị / index)
- repository: (.git repository) 


Cách commit đúng cách:
ví dụ:

TỐt
Fix: Tính đúng logic cho tổng tiền thanh toán
Feat: thêm chức năng reset password


Không tốt
update files
fix bugs and add new features


Ví dụ chị đang làm tính năng A
nhưng sếp lại yêu cầu chị qua fix bug của tính năng B
=> cần dùng đến branch (nhánh)

nhánh gốc là main
Chị làm tính năng A: chị sẽ tạo 1 nhánh mới từ nhánh main được gọi là nhánh A
    Sau khi phát triển + test xong tính năng A => sẽ gộp nhánh A vào trong nhánh main
Chị làm tính năng B: chị cũng sẽ tạo 1 nhánh từ nhánh main được gọi là nhánh B
    Sau khi phát triển + test xong tính năng B => sẽ gộp nhánh B vào trong nhánh main


Lệnh: 

//Lệnh tạo nhánh mới
git branch: kiểm tra xem nhánh hiện tại tên là gì

//lệnh kiểm tra nhánh
git branch ten-nhanh: tạo 1 nhánh mới

// chuyển nhánh
git switch ten-nhanh: chuyển qua nhánh khác (ưu tiên hơn)
git checkout ten-nhanh: cũng chuyển qua nhánh mới

// kết hợp tạo nhánh + chuyển nhánh 1 lần
git switch -c ten-nhanh: tạo và chuyển sang nhánh mới (ưu tiên hơn)
git checkout -b ten-nhanh: tạo và chuyển sang nhánh mới

//Gộp nhánh
git merge ten-nhanh-can-gop: Gộp nhánh
  - ví dụ chị đang ở nhánh main và gõ lệnh là:
      - git merge feature/header => nó sẽ lấy phần sao lưu của nhánh feature/header và đem gộp vào nhánh main

//Xoá nhánh
git branch -d ten-nhanh: xoá nhánh

lưu ý: option -d chỉ cho phép xoá khi nhánh đó đã được merge vào trong 1 nhánh khác
      Nếu thực sự không muốn dùng nữa và muốn xoá luôn không quan tâm nó đã được merge hay chưa thì sẽ sử dụng option -D
